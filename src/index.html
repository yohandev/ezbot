ezbot

ezbot is a plug-n-play robotics platform. You don't need (any) electronics or software knowledge to make a remote-controlled robot — just wire up motors and a battery, then connect your phone via Bluetooth.

This is ideal for classroom environments where the focus isn't EECS, though ezbot is flexible enough for ambitious students to add more peripherals and customize the firmware (see below).

Getting hardware

First you should order the PCBs. Note that this will take 2-3 weeks to arrive, so plan ahead. I recommend using JLCPCB (not affiliated, but the bill of materials is hard-coded to their parts inventory). Download the file here and follow the guide below:

The rest of the robot is up to you, but you should know a few things:

  • Battery The PCB is designed for 2S (7.4V) batteries (example). Also note that the boards come fully assembled if you followed the guide above, except for the battery connector; make sure to get some of those as well (example).
  • DC motors These are driven at the battery's voltage. Technically, this is beyond the rating for those common hobbyist motors, but it should be fine. Make sure to purchase the ones with header pin connectors (example).
  • Servo motors These are driven at 5V, so just about any hobbyist servo will work (example). On the board, the ground pins (brown/black wire) is the row closest to the antenna/USB port. Getting this backwards won't fry anything, it just won't work.

When you receive the boards, you'll need to flash the firmware. This can be done via this website (scroll down). If you're flashing 10+ boards, I recommend using the command-line interface (scroll down), which does this process semi-automatically (plug a board, wait, unplug, repeat). This requires node.js.


Lastly, you can 3D print the enclosure. The files are here.

Customizing the firmware

By default, only motors M0 and M1, and servo motor #0 are enabled. The board supports up to 4 DC motors + 4 servo motors, but you'll have to customize the firmware!


This is very simple, download Thonny, plug-in your board (make sure the original firmware is flashed, see below) and edit main.py.


You can also ask an LLM for help, it should be able to do this no-issue. It may be useful to give it the following link for context: https://github.com/yohandev/ezbot/tree/main/firmware/src

Flash the firmware

Mac / Linux:

export EZBOT=[CDN_URL]; curl -fsSL "$EZBOT/cli/install.sh" | sh

Windows (PowerShell):

$env:EZBOT='[CDN_URL]'; irm "$env:EZBOT/cli/install.ps1" | iex
